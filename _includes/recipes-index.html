{%- comment -%} ============================================================================== _includes/recipes-index.html — Rendu centralisé des cartes recette ============================================================================== Rôle - Produire le markup unique des cartes recette pour toutes les pages d’index (ex : index.html, tag-index.html), afin d’éviter toute divergence d’affichage. Entrées (paramètres d’include) - include.grouped (requis) • Tableau issu de `group_by: "recipe_group"`. • Chaque entrée contient : name (nom du groupe) + items (pages recettes). - include.hidden_tags (optionnel) • Array de tags “techniques” à masquer dans l’affichage des badges. Sortie (markup / data-* exposés) - .recipe-item[data-tested="true|false"] • Contrat requis par filter-tested.html (filtre “Seulement les recettes testées”). - a.recipe-card-link[data-title][data-subtitle][data-tested] • Contrat exploité par random-pick et/ou scripts de navigation. - article.recipe-card[data-tags][data-micros] • Métadonnées disponibles pour des filtres / recherches côté client. Contraintes - Stabilité : • Tri des groupes par g.name (affichage déterministe). • Tri des recettes par recipe_sort (ordre stable intra-groupe). - Tolérance aux données manquantes : • group name vide => groupe ignoré (évite ancres / sections invalides). • recipe_subtitle / badges_nutritionnels optionnels. - Accessibilité : • Hiérarchie : H3 = titre de groupe, H4 = titre de carte. • aria-label sur les listes de badges. Remarques - Le mapping tag/micronutriment → emoji + title est centralisé ici pour garantir une cohérence UI entre toutes les pages. - Le tag "gout-sucre" est un tag technique (classification), mais il est affiché volontairement sur les cartes via l’emoji 🍬 (si l’appelant ne le masque pas dans hidden_tags). ============================================================================== {%- endcomment -%} {%- comment -%} hidden_tags est optionnel : on sécurise pour éviter une erreur Liquid si l’appelant n’en fournit pas. IMPORTANT : - include.hidden_tags est en général un Array (split: ","). - on force un Array vide si absent pour éviter les pièges du `contains` sur string. {%- endcomment -%} {%- assign hidden_tags = include.hidden_tags | default: "" -%} {%- unless hidden_tags and hidden_tags.size > 0 -%} {%- assign hidden_tags = "" | split: "," -%} {%- endunless -%} {%- comment -%} Tri des groupes par nom : - Garantit un rendu stable quel que soit l’ordre de construction en amont. {%- endcomment -%} {%- assign grouped_sorted = include.grouped | sort: "name" -%}
